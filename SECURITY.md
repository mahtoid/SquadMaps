# Security policy

## Supported versions

Security fixes are applied to the live site at [squadmaps.com](https://squadmaps.com). This repository holds translations and community files — vulnerabilities in the web application itself should be reported as below.

## Reporting a vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Instead:

1. Use [GitHub private vulnerability reporting](https://github.com/mahtoid/SquadMaps/security/advisories/new) if enabled, **or**
2. Contact the maintainer via [Discord](https://discord.mahto.id/) and ask for a private channel.

Include:

- Description of the vulnerability
- Steps to reproduce
- Impact assessment (if known)
- Your contact details for follow-up

We aim to acknowledge reports within a reasonable timeframe and will keep you updated on remediation.

Thank you for helping keep SquadMaps safe for the community.
